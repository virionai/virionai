## The thesis

The durable output of AI work should not be a chat transcript.

It should be a portable artifact another person, model, system, auditor, or regulator can open later, verify offline, and continue without trusting the original runtime.

Virion builds the instruments underneath that shift: evidence substrates, portable work products, model harnesses, human workspaces, and evaluation loops.

## Open source

Three repositories carry the work.

- **[capsules-protocol](https://github.com/virionai/capsules-protocol)** — *Rust, JavaScript, Python.* The open protocol for portable, signed, AI-readable records of multi-actor work: spec, SDKs, CLI, verifier, and conformance harness.
- **[operators](https://github.com/virionai/operators)** — *TypeScript.* An offline, local-first investigative workspace: local Gemma via Ollama, visible ledger events, and Capsule handoff.
- **[researcher-brain](https://github.com/virionai/researcher-brain)** — *Logseq.* A second brain for research, built for the era when AI doesn't just retrieve knowledge — it produces it.

## Capsules

Capsule is an open protocol for portable AI work artifacts that carry content, state, provenance, participants, and a verifiable event ledger in a single `.capsule` file. Each surface below is a live example; the source sits beside it.

- **Open or verify a capsule** — [Reader](https://capsules.run/load/) · code: [capsules-protocol](https://github.com/virionai/capsules-protocol)
- **Build a command capsule** — [Operators demo](https://capsules.run/operator/) · code: [operators](https://github.com/virionai/operators)
- **Read the protocol spec** — [Protocol v0.6](https://capsules.run/protocol/v0.6/) · code: [capsules-protocol](https://github.com/virionai/capsules-protocol)
- **Check implementation signal** — [Conformance](https://capsules.run/conformance/) · code: [capsules-protocol](https://github.com/virionai/capsules-protocol)
